# ADR-009 Ver.1.0リリース範囲と将来拡張の分離

- 状態: 採用
- 日付: 2026-07-11

## 背景

初期の統合仕様は、最新マスタープロンプトの完成条件に加えて、専用通知センター、複数route UI、ダークモード、AI比較backtest、Beatへの全cron移行までVer.1.0必須としていた。一方、最新のユーザー指示は、実利用できる予想・AI独立・比較統合・買い目候補・結果分析・Docker/Windows・データ保護・運用手順を完成条件としている。

## 選択肢

1. 初期統合仕様の全拡張をVer.1.0まで実装する。
2. 最新マスタープロンプトの完成条件をVer.1.0とし、拡張機能を将来版へ分離する。
3. 未実装を隠して全項目を完了扱いにする。

## 採用

選択肢2を採用する。Ver.1.0必須は、データ取込・品質、Python予想、AI独立、比較統合、買い目候補、結果精算、条件別成績、履歴、ジョブ、画面内通知、設定可能な買い目上限、Docker/Windows、秘密管理、backup/restore、テスト、運用文書とする。

次は将来拡張とし、Ver.1.0の完成を妨げない。

- 専用Web通知センターと配信履歴。Ver.1.0は画面alert、ジョブ失敗理由、品質一覧、構造化logで代替する。
- UIの複数route化とダークモード。Ver.1.0は6領域navigation、responsive、keyboard、printを提供する。
- Python/AI/統合を横断する自動backtest。Ver.1.0はPythonロジック変更時の3か月backtestと、固定済み実データ受入を提供する。
- 既存cronの全Beat移行。Ver.1.0はCelery queue/Worker/Beat基盤と手動・Windows運用を提供する。
- 画面からのPython重み・AI prompt直接編集。Ver.1.0は版数固定、環境変数、画面の買い目条件を提供し、予想ロジック変更はreview/backtestを必須とする。

## 理由

- 最新ユーザー目標と完成条件を満たす。
- 自動投票、外部取得、モデル変更を安易に広げず安全性を優先できる。
- 未実装拡張を実装済みと扱わず、既知制限と追跡表で明示できる。

## 影響

- 統合仕様、追跡表、roadmap、既知制限を本ADRへ同期する。
- 将来拡張を実装するときは新migration、API、UI、回帰、移行手順を別版で追加する。
- Ver.1.0の既存DB/API/Artifactに破壊的変更はない。
