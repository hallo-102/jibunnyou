# ADR-007: 買い目候補と自動購入禁止境界

- 状態: 採用
- 決定日: 2026-07-11

## 背景

買い目作成では、Python案とAI統合案の違い、券種・組合せ・点数・金額・上限を確認できる必要がある。一方、候補保存や「購入済み」状態の記録を外部投票と混同すると誤購入につながる。

## 決定

1. 買い目は `python`、`ai_integrated`、`legacy_ai`、`manual` のsourceを区別して保存する。
2. AI統合案は固定済み統合output/hashをsourceとし、旧統合結果や馬同一性不一致から候補を作らない。
3. 対応券種はPhase 7で3連複とワイド、方式はformation、box、wheelとする。
4. 1点金額は100円以上・100円単位とし、最大点数、1レース上限、1日上限をBackendで検証する。
5. 重大不一致を含むAI統合案は既定で `review_required` とし、ユーザーの確認なしに購入候補へ進めない。
6. 全候補を `requires_confirmation=true`、`purchase_execution_enabled=false` とする。
7. `purchase_execution_enabled=false` はDB CHECK制約で固定し、自動投票経路を実装しない。
8. `purchased` statusは、外部で購入した事実をユーザーが手動記録する状態に限定する。
9. source hash、買い目rule版、warning code、組合せ、点数、金額、上限、理由を保存する。

## 結果

- Python案とAI統合案を同じ条件で比較できる。
- 点数・金額・上限超過・重大不一致を追跡できる。
- UIとDBの両方で「候補作成」と「外部購入」を分離できる。
