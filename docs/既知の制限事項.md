# Keiba AI Studio 既知の制限事項

- OpenAIのライブ独立分析は利用者のAPIキーとAPI Platform側の利用枠が必要。現在のキーは要求到達済みだが`insufficient_quota`のため、Billing設定後に利用可能となる。
- 既定Collectorはdry-runまたは明示承認sourceだけを実行する。規約未確認sourceを自動取得しない。
- JRA/IPAT等への自動投票機能は実装しない。購入状態は外部購入後の手動記録である。
- UIは6領域のURL・表示分離、ダークモード、route別取得最適化に対応し、運用・成績・AI比較・買い目・レース・共通ヘッダー・操作ツールバーを独立component化済み。API状態管理は引き続き共通親画面へ集約している。
- mobileは確認・軽操作を対象とし、大きなランキング表は表領域内で横scrollする。PCを主対象とする。
- Windows PowerShell以外のbackup/restore wrapperは未提供。DB dump自体はPostgreSQL custom formatで可搬。
- 外部HTML parserはsourceごとの利用規約とSchema承認後に追加する。曖昧な自動結合は行わない。
- `rating4_avg`の業務定義は既存Python資産に依存し、全面再定義は行っていない。
