# Keiba AI Studio 外部データ取得ポリシー

- 版数: 0.1
- 確認日: 2026-07-10
- 既定動作: `KEIBA_LEGACY_RUNNER_MODE=dry_run`
- 自動取得の承認済みsource: なし（`KEIBA_COLLECTOR_APPROVED_SOURCES=[]`）

## 1. 結論

既存取得スクリプトはCollector Adapterへ接続するが、コードが実行可能であることを利用許可とはみなさない。`execute`モードでは、現在の利用条件を人が確認したsource codeだけを`KEIBA_COLLECTOR_APPROVED_SOURCES`へ明示登録する。未登録sourceはCollectorが`blocked`で停止し、外部通信を開始しない。

robots.txtはアクセス制御の技術的意思表示の一部であり、著作権・契約上の利用許可そのものではない。本記録は法的助言ではなく、アプリの安全側の実行条件を定める運用記録である。

## 3. 実行を承認する前の必須チェック

1. 対象ドメイン、対象ページ、データ項目、利用目的、保存期間を記録する。
2. 最新の利用規約、個別サービス規約、robots.txt、公式API/提供データの有無を確認する。
3. 認証回避、CAPTCHA回避、アクセス制限回避を必要とする方式は不承認とする。
4. 1 sourceあたり同時実行1、最小間隔60秒、キャッシュ優先、最大3回の有限再試行を維持する。
5. HTML構造変更、Schema不一致、403/429、アクセス制限の兆候は自動再試行せず停止する。
6. 取得したRawのURL/ローカルsource、取得時刻、SHA-256、run ID、source code、信頼度を保存する。
7. 記事・コメント等は全文を長期保存せず、許可範囲の構造化要約と参照情報に限定する。
8. 承認者、確認日、次回確認期限を本書へ追記してからsource codeを環境変数へ登録する。

## 4. 有効化方法

確認済みsourceだけをJSON配列で指定する。

```dotenv
# 例示のみ。確認記録なしで値を追加しない。
KEIBA_LEGACY_RUNNER_MODE=execute
KEIBA_COLLECTOR_APPROVED_SOURCES=["SRC_JRA_003"]
```

設定変更後は`GET /api/v1/collection-sources`で`execution_approved=true`を確認し、検証環境で1対象・1回だけ実行する。HTTP回数を観測できない既存スクリプトは`request_count=0`とし、推測値を保存しない。

## 5. 即時停止条件

- 403、429、CAPTCHA、アクセス制限、利用条件変更の検知。
- HTML/CSV Schema変更、race_id・馬番・馬名の照合不一致。
- 同一sourceで連続失敗、想定外の大量リクエスト、キャッシュ無視。
- 取得物に不要な個人情報、認証情報、有料コンテンツ全文が含まれる。

停止後はsourceを承認配列から外し、`dry_run`へ戻し、CollectionRun・job log・Raw Manifestを確認する。古いSnapshotを最新として自動利用してはならない。
