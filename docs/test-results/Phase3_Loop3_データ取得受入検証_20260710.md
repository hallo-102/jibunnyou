# Phase 3 Loop 3 データ取得受入検証結果

- 実施日: 2026-07-10
- 対象: Collector queue、Raw→Normalized→Business、cache、有限再試行、照合・鮮度品質、取得UI
- Alembic revision: `0003_collector_layers`
- 外部取得モード: `dry_run`

## 実装結果

- 既存の出馬表、過去走、オッズ、結果スクリプトを専用`collector` Celery queueへ接続した。
- `collection_runs`と`collection_cache_entries`を追加し、source、対象、状態、試行数、cache hit、品質、失敗理由、Raw/Normalized Artifactを追跡可能にした。
- 読取専用の既存Excel/CSVをrun単位のRawへコピーし、コピー前後のSHA-256一致を確認して不変登録する。
- 正規化JSONをrun単位で保存し、同じtransaction系列でPostgreSQLのBusiness tableへ反映する。
- cache TTL、最小取得間隔60秒、最大3回の有限再試行を実装した。Schema・照合・設定エラーは再試行しない。
- OZZUの日付不一致、業務キー重複、馬番と馬名の矛盾を拒否し、曖昧な馬名fallbackを行わない。
- 複勝レンジは`odds_min`/`odds_max`へ保存し、中間値を捏造しない。
- 当日・発走前のオッズ鮮度を警告30分、critical 120分で検査する。
- 最新の出馬表/オッズ取得がrunning・failed・blocked・REDの場合、依存する予想/AI/買い目を停止する。
- UIへ取得状態、source、品質、cache、試行/再試行数、失敗理由、強制再取得、5秒pollを追加した。
- source別の規約確認記録を作成し、未承認sourceは`execute`開始前に`blocked`とする。

## 異常系自動テスト

- 一時Timeoutは10秒、60秒の順で再試行し、成功時のattempt/retry数を保存する。
- Schema/validationエラーは1回で停止し、sleepしない。
- 必須シート欠落、オッズ必須key欠落、日付不一致、重複、馬番/馬名矛盾を検出する。
- 当日オッズ121分経過をREDにする。
- date全体の別race REDが正常raceを誤停止しない一方、raceへ紐付け不能なsource errorは対象日全体を停止する。
- `execute`かつ未承認sourceは外部通信前に`blocked`となる。
