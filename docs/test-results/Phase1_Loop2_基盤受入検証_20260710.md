## 実コンテナ試験で検出・修正した不具合

### 1. 読取専用bind mountの重複

- 症状: `/workspace/data/raw`を読取専用mountした後、その配下へoddsをmountできずAPI起動失敗。
- 修正: Excel入力を`/workspace/data/raw/excel`へ分離し、兄弟ディレクトリとしてmount。
- 再試験: APIを含む全サービス起動成功。

### 2. Next.js standaloneのbind先

- 症状: Frontendがコンテナhostnameだけでlistenし、localhost healthcheckが接続拒否。
- 修正: runnerへ`HOSTNAME=0.0.0.0`と`PORT=3000`を設定。
- 再試験: Frontend healthcheck成功。

### 3. Nginxの旧コンテナIP保持

- 症状: API再作成後、Gatewayが旧IPへ接続し502を返却。
- 修正: Docker DNS `127.0.0.11`を設定し、変数形式upstreamで5秒ごとに再解決。
- 再試験: Gatewayを再起動せずAPI/Frontendを個別再作成し、全HTTP応答200。
