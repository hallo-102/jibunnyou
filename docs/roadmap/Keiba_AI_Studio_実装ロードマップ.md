## 2. 共通実装原則

- 既存の未コミット資産と既存Python予想を削除・全面置換しない。
- 1ループの変更を小さくし、対象、完了条件、テスト、ロールバックを明記する。
- Python予想ロジック変更時は3か月バックテストSkillの適用対象になる。UI/APIのみの変更では実行しない。
- 重大品質エラー、未承認AI、予算超過時は買い目を安全側へ停止する。
- `.env`、認証情報、実投票情報をGit・ログ・AIへ含めない。

## 3. Phase別計画

### M0：現状調査・設計統合

完了タスク:

- 27設計書の台帳、目的、関連、矛盾、重複、不足を整理する。
- 現行Backend、Frontend、Docker、データ領域、実行環境を調査する。
- 統合仕様、トレーサビリティ、ロードマップ、ADR、進捗を作る。

完了条件:

- 成果物のMarkdown lint相当確認が成功する。
- 設計書27件すべてが台帳に存在する。
- 現行Phase判定と次Phaseの優先作業が明確である。

ロールバック: Phase 0で新規追加した `docs/` ファイルのみ削除すれば実装へ影響しない。

### M1：開発基盤

タスク:

1. Composeを環境変数化し、Gatewayのみ18080を公開する。
2. Backendを本番依存でimport可能にし、Frontend Dockerfileをproduction起動へ直す。
3. `/health`、`/ready`、`/version`を分離する。
4. 構造化ログ、例外ハンドラ、request/trace IDを追加する。
5. Celery Worker/Beatを正しく登録し、pingタスクを検証する。
6. 最小のbackend/frontendテスト基盤を追加する。

完了条件:

- `docker compose up -d --build`が成功する。
- Gateway経由で画面とhealth/readinessを確認できる。
- DB、Redis、Worker状態を画面/APIで確認できる。
- 自動投票経路が存在しない。

### M2：DB・データ管理

タスク:

1. Alembicを導入し、既存モデルを初期migrationへ固定する。
2. 現行簡易モデルと統合仕様の差を段階migrationへ分ける。
3. Artifact、audit、version、AI独立分析、状態履歴の中核テーブルを追加する。
4. Excel/CSV/JSONのSchema・型・欠損・重複テストを作る。
5. バックアップ対象と復元契約を固定する。

完了条件:

- 空DBへmigration適用・downgrade/upgrade確認ができる。
- race_id・馬番・run_idの一意性がDBで守られる。
- 重要履歴が上書きされない。

### M3：競馬データ取得

タスク:

1. 既存取得スクリプトをCollectorキューへ接続する。
2. Raw→Normalized→Businessの保存を実装する。
3. 日付、race_id、馬番、馬名、取消、鮮度の品質ゲートを完成する。
4. キャッシュ、取得間隔、最大再試行を実装する。
5. 取得状況、失敗理由、再取得をUIへ出す。

完了条件:

- 指定日のレース・出走馬・オッズ・結果を保存できる。
- 不一致を危険に自動結合せず、手動確認へ回せる。

### M4：既存Python予想統合

タスク:

1. Input/Output Manifestを正式Schema化する。
2. subprocessの実行引数、作業フォルダ、終了コードを固定する。
3. 旧CLIとAdapterのGolden Master Fixtureを作る。
4. score、rank、確率、期待値、危険馬、根拠をDBとArtifactへ保存する。
5. 画面から非同期実行し、進捗・ログ・成果物を表示する。

完了条件:

- 同一入力で旧CLIと重要結果が許容差内一致する。
- 既存Excelシートを維持する。

### M5：生成AI独立予想

タスク:

1. `ai_independent_input_v1`を定義し、Python最終情報を禁止する。
2. Provider Adapterとキー未設定時Fallbackを作る。
3. 独立分析プロンプトとJSON Schemaを作る。
4. 独立結果をハッシュ付きで不変保存する。
5. 存在しない馬番、根拠なし、JSON不正、timeoutを検証する。

完了条件:

- 入力にPython最終順位・score・印・買い目が含まれないことを自動テストで保証する。
- AI独自順位、根拠、リスク、不明、自信度を保存できる。
- APIキーなしでもPython機能は継続する。

### M6：Python・AI比較と統合

タスク:

1. 独立結果固定後だけPython比較を許可する。
2. 順位差、理由差、一致・不一致、反対意見を構造化する。
3. 統合規則、補正上限、承認/Shadow Modeを実装する。
4. 信頼度、不確実性、順位範囲、データ品質を算出する。

完了条件:

- PythonとAIを並列表示し、独立性と統合根拠を追跡できる。
- 差分がない場合に差を捏造しない。

### M7：買い目作成

タスク:

1. Python案、AI統合案、手動案を分離する。
2. 3連複、ワイド等のプレビューと重複排除を実装する。
3. 1点、1レース、1日、最大点数の上限をBackendでも検証する。
4. 状態履歴、冪等性、最終確認を実装する。

完了条件:

- 上限超過、取消馬、データ不足、重大不一致で安全に停止する。
- 実投票を行わない。

### M8：結果照合・成績分析

タスク:

1. 結果・払戻の確定状態と同着・取消・失格を扱う。
2. 買い目精算を券種別にテストする。
3. Python/AI/統合の順位精度と買い目成績を比較する。
4. 外れ原因分類、回顧、期間・条件別KPIを実装する。

完了条件:

- 的中率、回収率、損益、最大連敗、最大ドローダウンを正しく算出する。

### M9：UI・UX全面改善

タスク:

1. 単一ページを共通Layout、ページ、Feature Componentへ分割する。
2. サイドバー、固定ヘッダー、下部鮮度表示を作る。
3. 結論カード、レースカード、上位8頭、詳細ドロワーを作る。
4. AI独立・比較・統合をタブ/段階表示する。
5. loading、empty、partial failure、error、再実行を統一する。
6. デスクトップ、タブレット、スマホ、印刷、キーボードを検証する。

完了条件:

- 最新改訂版45章の完成判定をすべてトレースできる。

### M10：テスト・品質改善

タスク:

1. Backend単体/API/DB/ファイル/異常系。
2. Frontend component/API/error/loading/empty。
3. Docker統合、E2E、回帰、障害、再起動。
4. バックアップ・復元、秘密情報、AI独立性。
5. 複数開催日の実データ受入。

完了条件:

- 重大・高優先度不具合0件。
- 主要自動テスト・回帰・E2Eが合格する。

### M11：運用・ドキュメント

タスク:

1. インストール、起動、停止、更新、バックアップ、復元、障害対応を整備する。
2. Web通知、Beat、pg_dump、保持処理を完成する。
3. API、DB、画面、テスト結果、既知制限、変更履歴を同期する。
4. 初心者が手順だけで起動・復旧できるか手順試験する。

完了条件:

- Ver.1.0の全完成条件とトレーサビリティが完了。
- 最終自己評価95点以上、かつ必須条件の欠落なし。
