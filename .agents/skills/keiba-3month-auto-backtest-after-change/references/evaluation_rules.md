# 評価ルール

## 集計単位

- 日別: `YYYYMMDD` ごとに集計する。
- 月別: `YYYYMM` ごとに集計する。
- 全体: 評価できた全レースを対象に集計する。

## レースの評価対象条件

- 予想Excelにレース単位の馬リストがある。
- 予想順位または予想スコアからランキング順を決められる。
- 結果Excel `data/master/racedata_results.xlsx` 側で同じレースを特定できる。
- 実際の1着、2着、3着、または各馬の着順を特定できる。

条件を満たさないレースは除外レースとして数える。除外理由は評価JSONに残す。

## 主な指標

| 指標 | 定義 |
| --- | --- |
| 対象日数 | 評価対象Excelが存在した日数 |
| 対象レース数 | 実績照合までできたレース数 |
| 除外レース数 | 予想または結果の不足で評価できなかったレース数 |
| 欠損馬数 | 予想上位馬が結果側で照合できなかった件数 |
| ランキング1位の馬券内率 | 予想1位馬が実着順3着以内だったレース割合 |
| ランキング1から3位の馬券内率 | 予想1から3位の各馬が3着以内だった割合 |
| ランキング1から5位の馬券内率 | 予想1から5位の各馬が3着以内だった割合 |
| TOP5完全捕捉率 | 実際の1から3着馬すべてが予想TOP5内に入ったレース割合 |
| 3連複想定的中率 | 指定した買い目対象馬内に実際の1から3着馬すべてが入った割合 |
| 回収率 | 的中時払戻合計 ÷ 購入想定金額 |
| 的中率のばらつき | 日別3連複想定的中率の標準偏差 |
| 回収率のばらつき | 日別回収率の標準偏差 |

## 3連複15点買い想定

既定では `--sanrenpuku-box-size 5` と `--sanrenpuku-bet-points 15` を使う。ヒット条件は「選択上位馬の中に実際の1から3着馬がすべて含まれること」。実際の買い目点数と違う運用にする場合は、引数で買い目数とBOX頭数を変更する。

## 回収率

`racedata_results.xlsx` の日付シートにある `払戻種別`, `組番`, `払戻金` から三連複払戻を探して計算する。払戻列が見つからない場合、回収率は `null` とし、判定では「要確認」材料にする。購入額は `bet_points * stake_yen` を1レース分として計算する。

## 穴馬捕捉と危険馬判定

- 穴馬を拾えた件数: 人気が7番人気以下の実着順3着以内馬を予想TOP5に入れた件数。
- 人気馬を危険馬として切れた件数: 人気が1から3番人気で実着順4着以下の馬を予想TOP5外にできた件数。

人気列が見つからない場合、この2指標は0または未計算として扱い、レポートで列不足を明記する。

## 判定

改善は、ランキング1位の馬券内率、TOP5完全捕捉率、3連複想定的中率、回収率のうち複数が改善し、対象レース数、除外レース数、欠損馬数に大きな悪化がない場合とする。

悪化は、軸馬精度または回収率が下がり、除外や欠損が増えた場合とする。

要確認は、的中率と回収率の方向が分かれる、対象レース数が減る、一部の日の大勝ちで全体が押し上がる、穴馬捕捉と軸馬安定性が両立していない場合とする。
